## 5. Live Currency Conversion via External API
Parent status: `[ ] Planned`

- [ ] Research and select a free, reliable currency conversion API (candidate: `https://api.frankfurter.app`).
- [ ] Assess rate limits, uptime guarantees, and offline/fallback behavior.
- [ ] Add a `CurrencyConversionService` that fetches live rates and caches them with a TTL.
  - Gracefully degrade to manual exchange rates when offline or when fetching fails.
- [ ] Integrate live rates into the Settings → Currency conversion flow as the default, with manual override still available.
- [ ] Add/update tests (mock the API; test online and offline/fallback paths).

Done definition:
- [ ] Currency conversion uses live rates by default when online, with a visible "last updated" timestamp.
- [ ] Conversion still works when network is unavailable (falls back to last cached rate or manual entry).

## 6. Reduce Rounding Errors in Currency Conversion
Parent status: `[ ] Planned`

- [ ] Note: If item 5 (live API) ships first, confirm whether live rates resolve this naturally before implementing manual precision fixes.
- [ ] Identify all conversion paths that lose precision on round-trip conversions.
- [ ] Increase stored decimal precision for exchange rates (support at least 6 significant decimal places).
- [ ] Display the computed inverse rate automatically when a user enters an exchange rate.
  - Example: entering `157.78 USD/JPY` shows `Inverse: 1 JPY = 0.00633839 USD`.
- [ ] Evaluate storing original-currency amounts as metadata to enable lossless "revert to original currency".
  - Implement if feasible without broad schema changes; defer to a follow-up otherwise.
- [ ] Warn users when an entered rate appears to be an imprecise inverse of a recent conversion.
- [ ] Add/update tests covering round-trip conversion precision and inverse-rate calculation.

Done definition:
- [ ] A USD → JPY → USD round-trip at realistic rates loses less than $1 on a $65,000 salary.
- [ ] Inverse rate is displayed to the user wherever an exchange rate is entered.

- [ ] Add/update tests (mock API; test population, manual override, and offline paths).

Done definition:
- [ ] Users can auto-populate tax lines with a single action from Pay Settings.

- [ ] Tax estimation works offline after initial fetch (cached rates used).
