## Goal
Add support for additional income sources beyond the base salary/hourly paycheck while preserving current budgeting, tax, and allocation behavior.

## Scope Decision for v0.5.0
- In scope (MVP): Fixed-amount other income entries with a frequency and taxable toggle.
- In scope (MVP): Income types such as Bonus, Commission/Tip, Rental Income, Retirement Withdrawal, and Other.
- In scope (MVP): Include these amounts in annualized and per-paycheck calculations and key summary views.
- Candidate expanded scope for v0.5.0: Percentage-of-gross bonus rules.
- Candidate expanded scope for v0.5.0: Automated tax withholding presets by income source type.

## Expanded Scope Preview (for v0.5.0 decision)

### A. Percentage-of-Gross Bonus Rules
- What this includes:
- Rule-based other-income entries where amount is computed as `% * baseGross`.
- Frequency support for annual, quarterly, and per-paycheck bonus-style entries.
- Optional cap/floor per period (for example: min $0, max $X).
- Optional scheduling window (start/end month in plan year).

- Proposed data model additions:
- [ ] Add `amountMode: 'fixed' | 'percent-of-gross'`.
- [ ] Add `percentOfGross?: number` (0-1000 range guard; final limit TBD).
- [ ] Add `appliesToGrossBasis?: 'base-pay-only' | 'base-plus-other-income-taxable'`.
- [ ] Add optional `activeMonths?: number[]` for quarterly/seasonal bonus behavior.

- Calculation behavior to define:
- [ ] Resolve `baseGross` from current pay settings before other-income additions.
- [ ] Compute percent-based entries deterministically in a fixed order.
- [ ] Document rounding strategy (round per line first vs round final aggregate).
- [ ] Prevent circular references by disallowing bonus calculations from including their own output.

- UI behavior to include:
- [ ] In Other Income form, add toggle between Fixed Amount and % of Gross.
- [ ] Show real-time computed preview per paycheck/month/year.
- [ ] Show warning when frequency + active months configuration yields sparse/irregular payouts.

- Testing required:
- [ ] Unit tests for percent math across pay frequencies.
- [ ] Unit tests for month-window behavior.
- [ ] Regression tests for mixed fixed + percent income stacks.

### B. Automated Tax Withholding Presets by Income Type
- What this includes:
- Preset withholding behavior per income type (for example: Bonus supplemental rate behavior).
- Optional "Auto" mode that proposes line impacts while preserving manual override.
- Transparent trace showing how withholding was computed.

- Proposed model/setting additions:
- [ ] Add `withholdingMode: 'manual' | 'auto' | 'hybrid'` at plan level or income-entry level.
- [ ] Add optional `withholdingProfileId` per income entry.
- [ ] Add internal mapping table for default profiles by income type.

- Calculation behavior to define:
- [ ] For taxable other income, compute withholding deltas separately from base paycheck taxes.
- [ ] Apply profile logic before final net-pay output.
- [ ] Keep current manual tax lines authoritative in manual mode.
- [ ] In hybrid mode, apply suggested withholding to dedicated "auto" tax lines only.

- UI behavior to include:
- [ ] Add withholding mode selector in Pay Details / Other Income context.
- [ ] Show an explain panel: source, profile, rate, taxable base, withheld amount.
- [ ] Provide one-click fallback to manual mode if users dislike auto behavior.

- Testing required:
- [ ] Unit tests for each profile path and mode (manual/auto/hybrid).
- [ ] Snapshot tests for explain panel outputs.
- [ ] Regression tests ensuring manual tax lines are not overwritten unexpectedly.

### v0.5.0 Decision Gate for Expanded Scope
- Include expanded scope in v0.5.0 only if:
- [ ] Core MVP (fixed-amount other income) is complete and stable first.
- [ ] Expanded-scope math is fully unit-tested and deterministic.
- [ ] Manual override behavior is explicit and tested.
- [ ] QA pass confirms no net-pay regressions in existing plans.

## Risks
- Risk: Tax logic coupling can create unintended net-pay regressions.
- Mitigation: Start with explicitly documented v0.5.0 tax handling and add focused tests.
- Risk: Scope creep from `% of gross` and withholding automation can delay release.
- Mitigation: Ship MVP first, then enable expanded scope only if the decision gate passes.
- Risk: User trust issues when auto withholding changes take-home pay unexpectedly.
- Mitigation: Keep manual mode available, show explainability details, and require explicit opt-in.

## Dependencies
- Depends on stable budget migration and validation paths in file storage.
- Depends on tab/search registration updates for discoverability.
